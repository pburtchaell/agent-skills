# Event Blocks

Create the right logistics blocks around any calendar event that has a location or requires travel. Covers concerts, festivals, movies, sporting events, appointments, and anything else with a time and place.

Shared conventions (calendars, work invite, simple titles, home base, no duplicates) live in the parent `SKILL.md`.

## Workflow
1. Identify the event: name, date/time, location, and type (concert, appointment, festival, movie, etc.).
2. Determine which blocks are needed based on event type (see Block Types below).
3. Create blocks working backwards from the event start time.

## Block Types

### Commute block
Created for any event with a location that requires travel.

Sizing:
- Red Rocks: 45 min
- Local Denver venues: 30 min (round up from 15)
- Appointments with a known address: calculate drive time from Whittier (2448 N Lafayette St) + 10 min buffer
- If drive time is unknown, ask or estimate conservatively

### Doors / gates block
Only for concerts and large venue events. Must be verified from the venue, ticket, or event listing. Do not assume or make up a doors time. This mainly matters for bigger venues like Red Rocks, stadiums, or amphitheaters. Skip for small local shows, movies, appointments, etc.

If doors exist, place the commute block before doors rather than before the event start.

### All-day visibility event
For concerts and special events (festivals, big outings), create an all-day event with the event name so it's visible in monthly calendar view. Not needed for routine appointments.

### Gap reminders
For events with a gap of more than 4 hours since the previous event:
- Weekday events: 2-hour reminder before the event
- Weekend events: also add a day-before (24-hour) reminder

## Calendar Routing
The event type sets the main event's calendar; every block for that event goes on the same one (see the block-placement rule in the parent `SKILL.md`).
- Concerts and special events → Family calendar
- Appointments and personal events → primary calendar

## Concert-Specific Rules
When a concert is on the Family calendar, ensure it has:
- An all-day event with the concert name (monthly view visibility)
- The timed event for the actual show
- A doors block only if there is a verified doors time
- A commute block before the show (or before doors if doors exist)

## Notes
- When a calendar review spots events missing blocks, add them.
- When verifying doors times for concerts, check the venue website or ticketing source. Never fabricate a doors time.
